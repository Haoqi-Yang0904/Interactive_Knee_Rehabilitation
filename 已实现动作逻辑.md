# knee_rehab_desktop_session.py 已实现动作逻辑

本文档对应当前版本的 `knee_rehab_desktop_session.py` 和 `rehab_actions/knee_desktop_library.py`。

## 总体说明

当前桌面训练程序使用 OpenCV 读取摄像头画面，并使用 MediaPipe Pose 的 `results.pose_landmarks` 做人体关键点识别。动作判断使用的是二维图像坐标：

- MediaPipe 归一化 2D 坐标：`landmark.x`、`landmark.y`
- 程序换算后的像素坐标：`[landmark.x * width, landmark.y * height]`
- 当前动作池不使用 `results.pose_world_landmarks`，也不使用深度信息

镜像显示只影响窗口中的画面和绘图，不改变识别逻辑。程序会先在原始摄像头帧上识别，再通过 `leg_for_display()`、`mirror_point_for_display()` 把点位映射到显示画面。

## 当前动作顺序

当前 `build_exercises()` 注册 8 个动作，顺序为：

1. 主动踝泵练习
2. 股四头肌等长收缩
3. 腘绳肌等长收缩
4. 仰卧位直抬腿
5. 侧抬腿练习
6. 后抬腿练习
7. 仰卧屈膝
8. 床上弯腿

已删除动作：`sandbag_knee_extension_press` / 沙袋压腿，`side_lying_hip_adduction` / 内收抬腿练习。

相关代码：


## 共用识别模块

### 关键点

所有动作共用 MediaPipe Pose 的下肢关键点：

- 髋：`LEFT_HIP` / `RIGHT_HIP`
- 膝：`LEFT_KNEE` / `RIGHT_KNEE`
- 踝：`LEFT_ANKLE` / `RIGHT_ANKLE`
- 足跟：`LEFT_HEEL` / `RIGHT_HEEL`
- 脚尖：`LEFT_FOOT_INDEX` / `RIGHT_FOOT_INDEX`

普通抬腿、等长收缩和屈膝动作使用 `hip-knee-distal`。`distal` 会在踝、足跟、脚尖中选择当前更稳定、可见性更好的小腿末端点。主动踝泵必须额外看到踝、足跟和脚尖，因为它要计算足部相对小腿的角度。

相关代码：

```python
def build_leg_points(...):
    hip = landmark_point(...)
    knee = landmark_point(...)
    distal_part = choose_distal_part(...)
    distal = landmark_point(...distal_part...)
```

### 侧面拍摄判断

侧面判断使用左右肩、左右髋的归一化 2D 坐标：

- 计算肩宽和髋宽
- 计算肩中点到髋中点的躯干长度
- 取 `max(肩宽, 髋宽) / 躯干长度`
- 如果比例大于 `SIDE_VIEW_MAX_TORSO_WIDTH_RATIO = 0.55`，提示用户转成侧面

这个判断主要用于体位提示，不单独决定计数。动作是否推进仍由关键点可见性、起始位稳定、角度达标、保持时间和回位条件共同决定。

相关代码：

```python
def side_view_feedback(landmarks) -> str | None:
    torso_width_ratio = max(shoulder_width, hip_width) / torso_length
    if torso_width_ratio > SIDE_VIEW_MAX_TORSO_WIDTH_RATIO:
        return "请把身体转成侧面，减少正面角度误差。"
```

### 共用角度

所有角度均基于 2D 图像坐标：

```python
def angle_at(point_a, vertex, point_c) -> float:
    ...

def knee_extension_angle(leg: LegPoints) -> float:
    return angle_at(leg.hip, leg.knee, distal)

def knee_flexion_angle(leg: LegPoints) -> float:
    return max(0.0, 180.0 - angle_at(leg.hip, leg.knee, leg.distal))

def leg_raise_angle_degrees(leg: LegPoints) -> float:
    vector = distal - leg.hip
    return degrees(atan2(upward, horizontal))
```

### 质量指标记录

训练结束时会在 `test_pose/knee_desktop_session_时间戳` 下保存 `summary.json` 和 `metrics_summary.txt`。不同动作记录的核心指标不同：

- 主动踝泵练习、股四头肌等长收缩、腘绳肌等长收缩：主要看完成次数；踝泵额外记录视觉达标循环数和最大踝泵角度幅度。

## 1. 主动踝泵练习

动作 ID：`ankle_pump`

执行模式：`ankle_hybrid`

拍摄体位：仰卧或坐位，患腿伸直，足部完整露出，最好能看到膝、踝、足跟和脚尖。

识别关键点：2D 膝、踝、足跟、脚尖。髋点仅用于辅助锁定下肢，不参与踝泵角度主判断。

关键角度：`ankle_foot_angle(leg)`，即小腿向量 `knee -> ankle` 与足部向量 `heel -> foot_index` 的 2D 夹角。

前置判断逻辑：

- 必须检测到人体和下肢
- 必须稳定看到踝、足跟、脚尖
- `AnkleAngleFilter` 会过滤跳变过大的踝角
- 连续达到 `ANKLE_READY_STABLE_FRAMES = 8` 帧后进入口令准备

动作过程判断：

- 一个循环包含勾脚保持 4 秒、放松 2 秒、下踩保持 4 秒、放松 2 秒
- 每帧更新本循环内的踝角最大值和最小值
- 循环结束时，如果样本数不少于 `ANKLE_MIN_CYCLE_SAMPLES = 8`，且角度范围大于等于 `target_movement_degrees = 12.0` 度，则认为视觉幅度达标

相关代码：

```python
def ankle_pump(target_units: int) -> Exercise:
    mode="ankle_hybrid"
    timer_phases=_ankle_phases()
    target_movement_degrees=12.0

def ankle_foot_angle(leg: LegPoints) -> float | None:

def run_timer_exercise(...):

    update_ankle_cycle_angle(filtered_angle)
    finish_ankle_cycle_quality()
```

## 2. 股四头肌等长收缩

动作 ID：`quadriceps_isometric`

执行模式：`timer`

拍摄体位：仰卧或坐位，患腿伸直，把膝盖后方轻轻压向床面。

识别关键点：2D 髋、膝、distal。distal 可为踝、足跟或脚尖。

关键角度：窗口显示 `knee_extension_angle(leg)` 作为参考，但动作计数不依赖肌肉形变识别。

前置判断逻辑：

- 检测到髋、膝和 distal
- 使用宽松的等长起始位稳定阈值 `ISOMETRIC_START_STABILITY_RATIO = 0.07`
- 起始位稳定 `ISOMETRIC_START_HOLD_SECONDS = 1.0` 秒后开始

动作过程判断：

- 主动收紧保持 5 秒，放松 2 秒
- 计数按节拍完成
- 如果下肢离开画面或起始位明显丢失，当前节拍暂停并提示回到起始位

相关代码：

```python
def quadriceps_isometric(target_units: int) -> Exercise:
    mode="timer"
    timer_phases=_isometric_phases()

def isometric_start_ready(leg, baseline_ankle):
    displacement = norm(leg.distal - baseline_ankle) / leg.leg_length
    return displacement < ISOMETRIC_START_STABILITY_RATIO

def run_timer_exercise(...):
    loose_isometric_start = exercise.id in LOOSE_ISOMETRIC_START_EXERCISE_IDS
```

## 4. 仰卧位直抬腿

动作 ID：`supine_straight_leg_raise`

执行模式：`visual`

拍摄体位：仰卧，患腿完全伸直，摄像头从侧方拍到髋、膝、踝。

识别关键点：2D 髋、膝、踝或 distal。

关键角度：

- 抬腿角：`leg_raise_angle_degrees(leg)`，即 `hip -> distal` 相对水平面的抬高角
- 膝伸直：`knee_extension_angle(leg)` 和 `knee_line_deviation_ratio(leg)` 用于提示膝盖弯曲；当前阈值已适度放宽为 `VISUAL_KNEE_LINE_MAX_RATIO = 0.12`，并用 `VISUAL_KNEE_STRAIGHT_MIN_DEGREES = 135.0` 辅助判断
- 目标抬腿角：`target_movement_degrees = 15.0`

前置判断逻辑：

- 检测到髋、膝和 distal
- 起始位稳定 `VISUAL_START_HOLD_SECONDS = 1.0` 秒
- 膝盖不能持续明显偏离髋-踝直线

动作过程判断：

- `raise`：抬腿角达到目标角度的 95%，并稳定 `VISUAL_TARGET_CONFIRM_SECONDS = 0.25` 秒
- `hold`：保持 `target_hold_seconds = 1.5` 秒
- `return`：抬腿角降到回位阈值以下，并稳定 `VISUAL_RETURN_HOLD_SECONDS = 0.5` 秒后计数

绘图逻辑：只画髋-膝、膝-distal 两条线；未达到目标角度或膝盖控制不合格时，腿段为黄色，目标角度虚线、distal 指向目标线终点的箭头、膝关节伸直旋转箭头为蓝色；达到目标角度且膝盖控制合格时，腿段变为绿色，并隐藏辅助线和辅助箭头。

相关代码：

```python
ANGLE_ONLY_VISUAL_EXERCISE_IDS = {
    "supine_straight_leg_raise",
    "side_lying_hip_abduction",
    "prone_hip_extension",
}

def run_visual_exercise(...):
    current_raise_angle = leg_raise_angle_degrees(leg)
    target_reached = current_raise_angle >= target_angle_degrees * 0.95

def draw_angle_only_leg_raise_guides(...):
    draw_dashed_line(frame, hip_px, target_px, ...)
```

## 5. 侧抬腿练习

动作 ID：`side_lying_hip_abduction`

执行模式：`visual`

拍摄体位：侧卧，患腿伸直向上抬。动作开始前语音提示“请侧躺在床面上，患侧腿在上，健侧腿在下，脸朝向镜头。”摄像头能看到髋、膝、踝，骨盆和躯干保持稳定。

识别关键点：2D 髋、膝、踝或 distal。

关键角度：

- 抬腿角：`leg_raise_angle_degrees(leg)`
- 目标抬腿角：`target_movement_degrees = 12.0`
- 膝伸直通过 `knee_line_deviation_ratio(leg)` 做质量提示

前置判断逻辑：

- 检测到髋、膝和 distal
- 自动选腿时优先追踪画面中更靠上的那条腿，用于对应“患侧腿在上”的侧卧姿势
- 起始位稳定 1 秒
- 膝盖不能持续明显弯曲或偏离直线

动作过程判断：

- 与仰卧位直抬腿共用角度型 `run_visual_exercise()` 逻辑
- 达到 12 度目标角的 95% 后进入保持
- 保持 `target_hold_seconds = 1.5` 秒
- 回到起始角度附近后计数

绘图逻辑：已移除距离/高度目标线，只保留髋-膝、膝-distal、髋关节出发的目标角度虚线，以及 distal 指向目标线终点的箭头。颜色规则与仰卧位直抬腿一致：未达标黄色，辅助提示蓝色，达标后绿色且隐藏辅助线。

相关代码：


## 7. 仰卧屈膝

动作 ID：`supine_knee_flexion`

执行模式：`flexion_visual`

拍摄体位：仰卧或坐位，侧面拍摄，完整看到髋、膝和小腿末端；先伸直患腿建立起始位。

识别关键点：2D 髋、膝、distal。distal 可为踝、足跟或脚尖。

关键角度：

- 屈膝角：`knee_flexion_angle(leg) = 180 - angle_at(hip, knee, distal)`
- 目标屈膝角：`target_flexion_degrees = 120.0`
- 目标保持时间：`target_hold_seconds = 6.0`

前置判断逻辑：

- 检测到髋、膝和 distal
- 起始位屈膝角不能超过 `FLEXION_START_MAX_DEGREES = 12.0`
- distal 相对起始位位移不能超过 `START_STABILITY_RATIO = 0.035`
- 起始位稳定 1 秒后进入口令

动作过程判断：

- `flex`：屈膝角达到目标的 95%，并通过角度稳定确认后进入保持
- `hold`：保持 6 秒；若角度掉到目标的 85% 以下则退回弯曲阶段
- `return`：伸直到屈膝角小于等于 `FLEXION_RETURN_MAX_DEGREES = 15.0`，并稳定 0.5 秒后计数

绘图逻辑：画髋-膝、膝-distal；已删除膝关节连接 distal 初始位置的基线。根据当前大腿和小腿方向计算目标 distal 位置，未达到目标屈膝角时画膝关节出发的蓝色目标屈膝虚线，并从当前 distal 指向目标 distal；达到目标屈膝角后腿段变为绿色，并隐藏辅助线和辅助箭头。

相关代码：

```python
def supine_knee_flexion(target_units: int) -> Exercise:

    target_flexion_degrees=120.0
    target_hold_seconds=6.0

def run_flexion_visual_exercise(...):
```

## 8. 床上弯腿

动作 ID：`bed_knee_flexion`

执行模式：`flexion_visual`

拍摄体位：仰卧，侧面完整看到髋、膝和脚踝；开始时膝盖不要求完全伸直。

识别关键点：2D 髋、膝、distal。

关键角度：

- 屈膝角：`knee_flexion_angle(leg)`
- 大腿上抬角：`thigh_raise_angle_degrees(leg)`，即 `hip -> knee` 相对水平面的夹角
- 记录指标：大腿最大上抬角、最大屈膝角、保持期大腿角波动、保持期屈膝角波动
- 目标保持时间：`target_hold_seconds = 10.0` 秒

前置判断逻辑：

- 检测到髋、膝和 distal
- 起始位保持稳定 1 秒
- 不强制要求膝盖完全伸直，因为该动作允许先以舒适姿势开始上抬大腿

动作过程判断：

- `lift`：先把大腿往上抬，达到个人最大限度并稳定后进入屈膝阶段；程序记录本回合和本次训练的大腿最大上抬角
- `flex`：保持大腿位置，再屈膝，把小腿往大腿方向收到最大限度；屈膝角稳定后进入保持阶段
- `hold`：在大腿上抬、屈膝的位置保持 10 秒；期间逐帧记录大腿上抬角和屈膝角，计算两个角度的波动范围，并以 `BED_HOLD_STABILITY_MAX_RANGE_DEGREES = 8.0` 度作为稳定性记录阈值
- `return`：大腿放下且膝盖回到接近伸直状态后计数

绘图逻辑：只画髋-膝大腿线和膝-distal 小腿线。`lift` 阶段显示髋关节水平参考虚线和大腿上抬方向箭头；`flex` 阶段显示小腿向大腿方向收的箭头；`hold` 阶段隐藏辅助线，只保留腿段。
