# knee_rehab_desktop_session.py 已实现动作逻辑

本文档对应 `knee_rehab_desktop_session.py` 和 `rehab_actions/knee_desktop_library.py` 当前实现。

## 总体识别说明

当前桌面训练程序使用 OpenCV 读取摄像头画面，并通过 MediaPipe Pose 的 `results.pose_landmarks` 做人体关键点识别。程序里的动作判断均使用 **二维图像坐标**：

- MediaPipe 输出的归一化 2D 坐标：`landmark.x`、`landmark.y`；
- 程序换算后的像素 2D 坐标：`[landmark.x * width, landmark.y * height]`；
- 当前 `knee_rehab_desktop_session.py` 没有使用 `results.pose_world_landmarks`，也没有使用三维空间坐标或深度信息。

需要注意：镜像显示只影响窗口中的画面和绘图，不改变识别逻辑本身。代码会先在原始摄像头帧上识别，再通过 `leg_for_display()` 和 `mirror_point_for_display()` 把点位翻到镜像画面上显示。

## 共用识别模块

### 关键点

所有下肢动作共用以下 MediaPipe Pose 关键点：

- 髋：`LEFT_HIP` / `RIGHT_HIP`
- 膝：`LEFT_KNEE` / `RIGHT_KNEE`
- 踝：`LEFT_ANKLE` / `RIGHT_ANKLE`
- 足跟：`LEFT_HEEL` / `RIGHT_HEEL`
- 脚尖：`LEFT_FOOT_INDEX` / `RIGHT_FOOT_INDEX`
- 肩和髋的左右点：用于判断是否接近侧面拍摄

### 侧面拍摄提示

侧面判断使用的是左右肩、左右髋的 **归一化 2D 坐标**。逻辑为：

- 计算肩宽：左肩到右肩的 2D 距离；
- 计算髋宽：左髋到右髋的 2D 距离；
- 计算躯干长度：肩中点到髋中点的 2D 距离；
- 取 `max(肩宽, 髋宽) / 躯干长度`；
- 若比例大于 `SIDE_VIEW_MAX_TORSO_WIDTH_RATIO = 0.55`，提示“请把身体转成侧面，减少正面角度误差。”

当前实现中，这个检查主要是 **提示性前置判断**，用于提醒拍摄体位不理想；它不会单独硬性阻断所有动作计数。动作是否推进仍主要取决于关键点可见性、起始位稳定、角度达标、保持时间和回位条件。

相关代码：

## 1. 主动踝泵练习

动作 ID：`ankle_pump`

执行模式：`ankle_hybrid`

拍摄体位：

- 仰卧或坐位；
- 患腿伸直；
- 摄像头尽量拍到膝、踝、足跟和脚尖；
- 如果身体过于正面，窗口会提示转成侧面。

识别关键点：

- 2D 髋、膝：用于锁定腿侧和下肢结构；
- 2D 踝：用于小腿方向；
- 2D 足跟、脚尖：用于足部方向；
- 踝、足跟、脚尖必须清晰，否则无法计算踝角。

关键角度：

- `ankle_foot_angle`：小腿向量 `knee -> ankle` 与足部向量 `heel -> foot_index` 的 2D 夹角；
- 单个完整循环内记录最小角度和最大角度；
- `angle_range = cycle_max_angle - cycle_min_angle`；
- 当前目标活动幅度：`target_movement_degrees = 12.0` 度。

前置判断逻辑：

- 必须检测到人体和下肢；
- 必须能稳定看到踝、足跟和脚尖；
- `AnkleAngleFilter` 会过滤跳变过大的踝角；
- 连续达到 `ANKLE_READY_STABLE_FRAMES = 8` 帧后，才进入“准备，开始”。

动作过程判断逻辑：

- 一个循环包含 4 个阶段：勾脚保持 4 秒、放松 2 秒、下踩保持 4 秒、放松 2 秒；
- 每帧更新当前循环内的踝角范围；
- 循环结束时，如果样本数不少于 `ANKLE_MIN_CYCLE_SAMPLES = 8` 且角度范围大于等于 12 度，则认为视觉幅度确认；
- 计数本身按节拍推进，视觉逻辑用于质量提示和警告。

## 2. 股四头肌等长收缩

动作 ID：`quadriceps_isometric`

执行模式：`timer`

拍摄体位：

- 仰卧或坐位；
- 患腿伸直；
- 膝盖后方轻轻压向床面；
- 侧面拍摄更适合判断下肢是否稳定。

识别关键点：

- 2D 髋、膝、远端点；
- 远端点可为踝、足跟或脚尖，由当前可见性自动锁定。

关键角度：

- 窗口显示 `angle_at(hip, knee, distal)`，但对该动作标注为“仅参考”；
- 当前等长收缩动作不直接用肌肉收缩幅度判断质量，因为单摄像头难以可靠识别肌肉收缩。

前置判断逻辑：

- 检测到髋、膝、远端点；
- 建立远端点起始位；
- 使用较宽松的等长起始位稳定阈值：`ISOMETRIC_START_STABILITY_RATIO = 0.07`；
- 起始位稳定 `ISOMETRIC_START_HOLD_SECONDS = 1.0` 秒后开始。

动作过程判断逻辑：

- 阶段为“主动收紧保持 5 秒”和“放松 2 秒”；
- 计数按节拍完成；
- 如果训练中下肢离开画面，当前节拍会暂停，并提示回到起始位；
- 重新识别后继续当前节拍。

## 4. 沙袋压腿

动作 ID：`sandbag_knee_extension_press`

执行模式：`timer`

拍摄体位：

- 侧面拍摄；
- 脚跟垫软物使腿悬空；
- 沙袋压在膝关节上；
- 髋、膝、小腿末端需要可见。

识别关键点：

- 2D 髋、膝、远端点；
- 远端点优先踝，也可在足跟、脚尖中自动选择稳定点。

- `angle_at(hip, knee, distal)`；
- 起始位要求膝关节内角不小于 `KNEE_STRAIGHT_MIN_DEGREES = 150.0` 度。

前置判断逻辑：

- 检测到髋、膝、远端点；
- 膝盖需要伸直；
- 远端点相对基线位移需要小于 `START_STABILITY_RATIO = 0.035`；
- 起始位稳定 1 秒后开始。

动作过程判断逻辑：

- 单阶段保持：默认 15 分钟；
- 计时过程中如果下肢离开画面，会暂停并提示回到起始位；
- 完成保持时长后计为 1 次。

## 5. 仰卧位直抬腿

动作 ID：`supine_straight_leg_raise`

执行模式：`visual`

拍摄体位：

- 仰卧；
- 患腿完全伸直；
- 摄像头从侧方拍到髋、膝、踝；
- 起始位让脚踝或远端点停稳 1 秒。

识别关键点：

- 2D 髋、膝、远端点；
- 如果踝可见，优先用踝参与膝伸直判断；
- 使用起始位髋和远端点建立运动基线。

关键角度和距离：

- 膝伸直角：`knee_extension_angle(leg)`；
- 膝偏离直线比例：`knee_line_deviation_ratio(leg)`；
- 抬高角：当前 `hip -> distal` 向量相对起始基线的夹角；
- 离床距离：远端点相对起始基线的垂直位移，按用户输入的腿长换算为厘米；
- 目标：约 15 度，`visual_up_threshold = 0.14`，`visual_down_threshold = 0.05`。

前置判断逻辑：

- 必须检测到人体和下肢关键点；
- 起始位远端点位移小于 0.03，稳定 `VISUAL_START_HOLD_SECONDS = 1.0` 秒；
- 如果膝盖偏离直线比例超过 `VISUAL_KNEE_LINE_MAX_RATIO = 0.08` 并持续 `0.9` 秒，会提示膝盖伸直。

动作过程判断逻辑：

- `setup`：建立起始位；
- `raise`：抬到目标线，目标达到条件为抬高比例或抬高角达到目标的 95%，并确认 `VISUAL_TARGET_CONFIRM_SECONDS = 0.25` 秒；
- `hold`：保持 `target_hold_seconds = 1.5` 秒，低于保持阈值超过宽限时间会退回抬起阶段；
- `return`：远端点回到起始位附近，稳定 `VISUAL_RETURN_HOLD_SECONDS = 0.5` 秒后计数。

## 6. 床上弯腿

动作 ID：`bed_knee_flexion`

执行模式：`flexion_visual`

拍摄体位：

- 仰卧或坐位；
- 侧面完整看到髋、膝和小腿末端；
- 先伸直患腿建立起始位。

识别关键点：

- 2D 髋、膝、远端点；
- 远端点可以是踝、足跟或脚尖；
- 起始位记录 `baseline_distal` 和 `baseline_hip`。

关键角度：

- 屈膝角：`knee_flexion_angle = 180 - angle_at(hip, knee, distal)`；
- 当前目标屈膝角：`target_flexion_degrees = 70.0` 度；
- 回位要求屈膝角小于等于 `FLEXION_RETURN_MAX_DEGREES = 15.0` 度；
- 起始位要求屈膝角不大于 `FLEXION_START_MAX_DEGREES = 12.0` 度。

前置判断逻辑：

- 检测到髋、膝和远端点；
- 起始位屈膝角不能超过 12 度；
- 远端点相对起始位位移不能超过 `START_STABILITY_RATIO = 0.035`；
- 起始位稳定 1 秒后开始。

动作过程判断逻辑：

- `setup`：伸直并建立起始位；
- `flex`：慢慢弯曲，屈膝角达到目标的 95% 且角度稳定后进入保持；
- `hold`：保持 10 秒；若角度掉到目标的 85% 以下，退回弯曲阶段；
- `return`：伸直回位，屈膝角小于等于 15 度并稳定 0.5 秒后计数。

当前 `bed_knee_flexion` 的 `target_lift_degrees` 为 0，因此不会进入额外的 `lift` 上抬阶段；`run_flexion_visual_exercise()` 保留了带上抬角的扩展逻辑，用于以后可能增加“先上抬再屈膝”的动作。

## 7. 侧抬腿练习

动作 ID：`side_lying_hip_abduction`

执行模式：`visual`

拍摄体位：

- 侧卧；
- 患腿伸直向上抬；
- 摄像头看到髋、膝、踝；
- 骨盆和躯干尽量稳定。

识别关键点：

- 2D 髋、膝、远端点；
- 使用起始位髋到远端点的方向作为基线；
- 使用远端点相对基线的垂直位移和抬高角判断动作幅度。

关键角度和距离：

- 膝伸直角和膝偏离直线比例用于防止膝盖明显弯曲；
- 目标抬高角：约 12 度；
- `visual_up_threshold = 0.12`，`visual_down_threshold = 0.05`；
- 保持时间：1.5 秒。

前置判断逻辑：

- 髋、膝、远端点可见；
- 起始位稳定 1 秒；
- 膝盖不能持续明显偏离直线。

动作过程判断逻辑：

- 与仰卧位直抬腿共用 `run_visual_exercise()`；
- `raise` 达到目标线并稳定 0.25 秒；
- `hold` 保持 1.5 秒；
- `return` 回到起始位附近并稳定 0.5 秒后计数。
